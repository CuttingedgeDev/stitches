title: Breakpoints
description: Set-up reusable responsive breakpoints.
---

Define your media at-rules in the `conditions` object. You can add as many as you need.

```jsx line=2-7
export const { styled, css } = createCss({
  conditions: {
    bp1: '@media (min-width: 640px)',
    bp2: '@media (min-width: 768px)',
    bp3: '@media (min-width: 1024px)',
  },
});
```

> Note: Choose the naming convention you prefer. We recommend either <Code>bp1</Code>, <Code>bp2</Code>, <Code>bp3</Code> etc. or <Code>sm</Code>, <Code>md</Code>, <Code>lg</Code> etc.

#### Responsive variants

Once configured, you can apply different variants at different [conditions](/docs/breakpoints).

In the example below, we apply the `violet` color variant at the `bp2` condition.

```jsx line=4-9,12-16
const Button = styled('button', {
  // Base styles

  variants: {
    color: {
      __violet__: {},
      gray: {},
    },
  },
});

() => (
  <Button
    color={{
      __bp2__: __'violet'__
    }}
  >
    Button
  </Button>
);
```

> Note: If you're using TypeScript, your variants will be typed.

#### Setting an initial variant

You must use the <Code>initial</Code> breakpoint to declare the initial variant before any breakpoints are applied.

In the example below, we apply `gray` initially, then the `violet` color variant at the `bp2` condition.

```jsx line=4-9,12-16
const Button = styled('button', {
  // Base styles

  variants: {
    color: {
      violet: {},
      __gray__: {},
    },
  },
});

() => (
  <Button
    color={{
      __initial__: __'gray'__
      bp2: 'violet'
    }}
  >
    Button
  </Button>
);
```

#### Using breakpoints in the style objects

```jsx line=4-8
const Button = styled('button', {
  // Base styles

  when: {
    bp1: {
      // Styles for bp1
    },
  },
});
```

> Note: In most cases, we do not recommend applying responsive breakpoints inline. Ideally, your component styles should be immutable.
