title: Responsive Styles
description: How to configure breakpoints and apply variants responsively.

#### Configure your breakpoints

Define your breakpoints in the `conditions` object. Add as many breakpoints as you need.

```jsx line=2-7
export const { styled, css } = createCss({
  conditions: {
    bp1: '@media (min-width: 640px)',
    bp2: '@media (min-width: 768px)',
    bp3: '@media (min-width: 1024px)',
    bp4: '@media (min-width: 1280px)',
  },
});
```

> Note: Choose whichever naming convention you prefer. We recommend either <Code>bp1</Code>, <Code>bp2</Code>, <Code>bp3</Code>, <Code>bp4</Code> etc. or <Code>sm</Code>, <Code>md</Code>, <Code>lg</Code>, <Code>xl</Code> etc.

#### Responsive variants

Once configured, you can apply different variants at different [conditions](/docs/breakpoints). You must use the <Code>initial</Code> breakpoint to declare the initial variant before any breakpoints are applied.

In the example below, we apply `gray` initially, then the `violet` color variant at the `bp2` condition.

```jsx line=12-16
const Button = styled('button', {
  ...styles,

  variants: {
    color: {
      violet: { ...violetStyles },
      gray: { ...grayStyles },
    },
  },
});

() => (
  <Button
    color={{
      initial: 'gray',
      bp1: 'violet',
    }}
  >
    Button
  </Button>
);
```

> Note: If you're using TypeScript, your variants will be typed.

#### Using breakpoints in the style objects

```jsx line=4-9
const Button = styled('button', {
  height: '35px'

  // apply styles to the `bp1` breakpoint
  when: {
    bp1: {
      height: '45px'
    }
  }
});
```

> Note: In most cases, we do not recommend applying responsive breakpoints inline. Ideally, your component styles should be immutable.
