Both Stitches and Styled Components are CSS-in-JS libraries. Migrating from Styled Components doesn't involve structural or tooling changes to your code base. Most of the differences are in the styling API.

In this post I'll highlight the main differences between Styled Components and Stitches. I'll show you how you can convert your existing Styled Components to use Stitches instead.

---

#### Importing `styled` Function

In Styled Components, you can import the `styled` function directly from `styled-components.

In Stitches, you import `styled` from where you've [created the config file](/docs/installation#create-your-config-file).

```jsx
// Styled Components
import { styled } from 'styled-components';

// Stitches
import { styled } from 'stitches.config.js';
```

#### Writing CSS in oOject Syntax

In Styled Components, you can write CSS in template strings.

In Stitches, you have to use the [object style](/docs/styling#object-syntax) sytax.

```jsx
// Styled Components
const Button = styled.button`
  color: red;
  &:hover {
    color: black;
  }
`;

// Stitches
const Button = styled('button', {
  color: 'red',
  '&:hover': {
    color: 'black',
  },
});
```

#### Interpolating Props

In Styled Components, you can interpolate the component's props to conditionally set styles.

In Stitches, prop interpolation is not supported. This is mainly for performance reasons. Instead Stitches introduces the concept of [variants](/docs/variants), which should be able to replace prop interpolation in most cases.

```jsx
const Button = styled.button`
  color: ${(props) => (props.variant === 'black' ? 'black' : 'red')};
`;

const Button = styled('button', {
  color: 'red',
  variants: {
    variant: {
      black: {
        color: 'black',
      },
    },
  },
});

() => <Button variant="red">Button</Button>;
```

#### Adding Theme and Tokens

In Styled Components, you can add a theme via the `<ThemeProvider/>`. The theme gets injected to every component, and you can access it via prop interpolation.

In Stitches, you can [define tokens](/docs/tokens) in the config file and seamelessly consume them in the Style Objects.

```jsx
// Styled Components
import { ThemeProvider } from 'styled-components';

() => (
  <ThemeProvider
    theme={{
      tokens: {
        colors: {
          $red500: 'tomato',
        },
        space: {
          $1: '5px',
          $2: '10px',
        },
      },
    }}
  >
    <App />
  </ThemeProvider>
);

const Button = styled.button`
	color: ${props} => props.theme.colors.$red500;
	margin: ${props} => `${props.theme.space.$1} ${props.theme.space.$2}`
`;

// Stitches
const { styled } = createStyled({
  tokens: {
    colors: {
      $red500: 'tomato',
    },
    space: {
      $1: '5px',
      $2: '10px',
    },
  },
});

const Button = styled('button', {
	color: '$red500',
	margin: "$1 $2"
});
```

#### Server-Side Rendering

In Styled Components, you create `ServerStyleSheet` or the `StyleSheetManager` depending on your needs.

In Stitches, you use the `getStyles` function. Learn more about [server-side rendering](/docs/server-side-rendering).
