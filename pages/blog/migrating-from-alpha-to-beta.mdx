You can use this guide to help with the migration from Stitches alpha to beta.

---

### Breaking changes

The API is mostly the same, with a few small breaking changes, listed below.

#### Default export

The creator function is now a default export.

`@stitches/core`

```diff
- import { createCss } from '@stitches/core';
+ import createCss from '@stitches/core';
```

`@stitches/react`

```diff
- import { createStyled } from '@stitches/react';
+ import createCss from '@stitches/react';
```

#### Independent functions

Functions such as `global`, `keyframes` and `theme` used to belong to the `css` object. Now they're independent.

```diff
- import { css, styled } = createCss()
+ import { css, styled, global, theme, keyframes, getCssString } = createCss()
```

#### Theme configuration

The `tokens` key in the configuration has been renamed to `theme`.

```diff
createCss({
- tokens: {}
+ theme: {}
});
```

#### Breakpoints configuration

A couple of things here:

- The `breakpoints` key in the configuration has been renamed to `conditions`.
- A `condition` is now just a key-value pair. No more function.

```diff
createCss({
- breakpoints: {
+ conditions: {
-   bp1: (rule) => `@media (min-width: 640px) { ${rule} }`,
+   bp1: '@media (min-width: 640px)'
  }
});
```

#### Using tokens

To use tokens you now must prefix them with a `$`.

```diff
const Button = styled('button', {
- color: 'red'
+ color: '$red'
});
```

#### Chaining pseudo classes and elements

If you want to chain pseudo-classes or pseudo-elements you now must use the `&` qualifier.

```diff
const Button = styled('button', {
- ':hover': {}
+ '&:hover': {}
- '::before': {}
+ '&::before': {}
});
```

#### Responsive styles

Responsive styles are now nested in a `when` key.

```diff
const Button = styled('button', {
- bp1: {}
+ when: {
+   bp1: {}
+ }
})
```

#### Compound variants

This is now a special `compoundVariants` array in the style object.

```diff
const Button = styled('button', {
  variants: {
    color: {
      violet: {},
      gray: {},
    },
    appearance: {
      outline: {}
    }
  },
+ compoundVariants: [{
+   color: 'violet',
+   appearance: 'outline',
+   css: {
+     color: 'blueViolet',
+     borderColor: 'darkviolet',
+     '&:hover': {
+       color: 'white',
+     },
+   }
+ }]
})

- Button.compoundVariant({
-  color: 'violet',
-  appearance: 'outline',
- }, {
-  color: 'blueViolet',
-  borderColor: 'darkviolet',
-  ':hover': {
-    color: 'white',
-  },
- })
```

#### Server-side rendering

A couple of things here:

- The `getStyle` function has been renamed to `getCssString`.
- The `getCssString` function receives no arguments, just call it and it'll render your styles as a string.

```diff
- const { result, styles } = css.getStyles(() => renderSomething());
+ const styles = getCssString()
```

#### Applying classnames

When using the `css` function, you now must call the stitches component to apply the class.

```diff
const button = css({})

() => (
- <Button className={button} />
+ <Button className={button()} />
)
```

#### Only one & per selector

You can only use one `&` (one ampersand) per selector. If you were using `&&` (two ampersands) to increase the specificity of a rule check that it's still necessary as we've fixed lots of specificity issues.

```diff
const Button = styled('button', {
- '&&': {}
})
```

If for some reason you must increase the specificity, we recommend doing it via an the class attribute selector:

```diff
const Button = styled('button', {
- '&&': {}
+ '&[class]': {}
})
```

### New features

We've also introduced some exciting new features, here's an overview.

#### Reference tokens on the same scale

You can now create semantic tokens by referencing tokens on the same scale. As always, use the `$` to reference a token.

```diff
createCss({
  theme: {
    colors: {
      gray500: 'hsl(206,10%,76%)',
+     primary: '$gray500'
    }
  }
})
```

#### Choose your theme class

You can now define the class to be generated when creating additional themes.

```diff
+ const darkTheme = theme('dark-theme', {})
```

#### Applying theme

In `@stitches/react` to get the generated class name of a theme, you need to use the `className` static property.

```diff
const darkTheme = theme({})

() => (
- <div className={darkTheme} />
+ <div className={darkTheme.className} />
)
```

#### Default variants

You can now set default variants in `css` and `styled` functions via a `defaultVariants` key in the style object.

```diff
const Button = styled('button', {
  variants: {
    color: {
      violet: {},
      gray: {},
    },
    appearance: {
      outline: {}
    }
  },
+ defaultVariants: {
+   color: 'violet',
+ }
})
```

#### Customisable property mapping

You can now override the default mapping of property to scale via the `themeMap` configuration.

```diff
createCss({
  themeMap: {
+   height: 'space',
+   width: 'space'
  }
})
```

If you wish to merge with the default mapping, you can import and spread `defaultThemeMap`. This is available from `@stitches/core` and `@stitches/react`.

```diff
import { defaultThemeMap } from '@stitches/core'

createCss({
  themeMap: {
+   ...defaultThemeMap,
+   height: 'space',
+   width: 'space'
  }
})
```

#### Pick a token from a specific scale

You can now pick a token from a specific scale instead of the mapped one. For example, by default `marginTop` is mapped to the `space` scale. But it's possible to pick a token from the `size` scale, by prefixing it with the scale name.

```diff
const Button = styled('button', {
+ marginTop: '$sizes$4'
})
```

#### Locally scoped tokens

You can now create a token directly within a style object by prefixing it with `$$` (two dollar signs).

```diff
const Button = styled('button', {
+ $$borderWidth: '1px',
+ boxShadow: '0 0 0 $$borderWidth red'
})
```

Just as you can [pick a token from a specific scale](#pick-a-token-from-a-specific-scale) in a property value, you can in a token:

```diff
+ $$borderWidth: '$borderWidths$1'
```

#### Support for `@import` rule

You can now use `@import` rules within the `global` function.

```diff
const globalStyles = global({
+ '@import': 'custom.css'
})
```

#### Support for `@font-face` rule

You can now use `@font-face` rules within the `global` function.

```diff
const globalStyles = global({
+ '@font-face': {
+   fontFamily: 'CustomFont',
+   src: 'local("CustomFont"), url("CustomFont.woff2")'
+ }
})
```

#### Support for `@supports` rule

You can now use `@supports` rules.

```diff
const globalStyles = global({
+ '@supports (display: grid)': {
+   body: {
+     display: grid
+   }
+ }
})

const Grid = styled('div', {
+ '@supports (display: grid)': {
+   display: grid
+ }
})
```

#### Theme object

The `theme` is now an object containing useful token data.

For example, given the following default `theme` and `dark` themes:

```jsx
const { theme } = createCss({
  theme: {
    colors: {
      primary: 'black',
    },
  },
});

const dark = theme({
  colors: {
    primary: 'white',
  },
});
```

You can use the returned theme objects to read the tokens, like so:

```diff
// default theme
+ theme.colors.primary.value; // black
+ theme.colors.primary.token; // primary
+ theme.colors.primary.scale; // colors
+ theme.colors.primary.variable; // --colors-primary

// dark theme
+ dark.colors.primary.value; // white
+ dark.colors.primary.token; // primary
+ dark.colors.primary.scale; // colors
+ dark.colors.primary.variable; // --colors-primary
```

#### Core variant support

The `@stitches/core` package now supports `variants`, as well as `defaultVariants` and `compoundVariants`.

```diff
const button = css('button', {
+  variants: {
+   color: {
+     violet: {},
+     gray: {},
+   },
+ }
})

() => (
+ <Button className={button({ color: 'violet' })} />
)
```

#### Core override support

The `@stitches/core` package now supports overrides via a `css` prop.

```diff
const button = css('button', {})

() => (
+ <Button className={button({ css: { display: 'none' } })} />
)
```

### Typescript

We've introduced some type helpers which can be useful when using Stitches components.

#### Abstract styles

You can abstract style objects while still get typing with by creating with the help of the `StitchesCss` generic.

> This is available in both `@stitches/core` and `@stitches/react`.

```diff
// stitches.config.ts
+ import createCss, { StitchesCss } from '@stitches/core';


+ const stitchesConfig = createCss({
  theme: {
    colors: {
      gray500: 'hsl(206,10%,76%)',
    }
  }
});


+ const { css } = stitchesConfig


+ export type CSS = StitchesCss<typeof stitchesConfig>;


+ const sharedStyles: CSS = {
+ 	backgroundColor: '$gray500'
+}
```

#### Extract variants

You can extract the variants of a component with the `StitchesVariants` type.

> This is available in both `@stitches/core` and `@stitches/react`.

```diff
+ import { StitchesVariants } from '@stitches/core';


+ type ButtonVariants = StitchesVariants<typeof button>;


const button = css('button', {
   variants: {
    color: {
      violet: {},
      gray: {},
    },
  }
})
```
